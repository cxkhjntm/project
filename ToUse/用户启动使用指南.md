## 快速启动

如果您已安装好环境，可按以下步骤快速启动：

```bash
# 1. 进入项目目录
cd project

# 2. 启动后端（新终端窗口）
cd backend
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# 编辑 .env 填入 ENCRYPTION_KEY（见下方说明）
alembic upgrade head
uvicorn app.main:app --reload --port 8000

# 3. 启动前端（另一个终端窗口）
cd frontend
npm install
npm run dev

# 4. 访问 http://localhost:5173
```

## 详细启动步骤

### 第一步：启动后端服务

#### 1.1 进入后端目录

```bash
cd project/backend
```

#### 1.2 创建 Python 虚拟环境

```bash
# 创建虚拟环境
python -m venv .venv

# 激活虚拟环境
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate
```

激活后，终端提示符会显示 `(.venv)` 前缀。

#### 1.3 安装 Python 依赖

```bash
pip install -r requirements.txt
```

如果安装缓慢，可使用国内镜像源：

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

#### 1.4 配置环境变量

```bash
# 复制环境变量模板
# Windows:
copy .env.example .env
# macOS/Linux:
cp .env.example .env
```

编辑 `.env` 文件，**必须配置** `ENCRYPTION_KEY`：

```env
# 生成加密密钥（在终端执行）
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

将生成的密钥填入 `.env` 文件：

```env
ENCRYPTION_KEY=生成的密钥字符串
```

> **重要**：
> - 此密钥用于加密 LLM API 密钥，请妥善保管。如果丢失，已保存的 API 密钥将无法解密。
> - 密钥必须是有效的 Fernet 格式（32 字节 URL-safe Base64 编码，通常为 44 个字符，以 `=` 结尾）。
> - 系统会优先使用 `.env` 中的 `ENCRYPTION_KEY`；如果未配置或格式无效，会自动从 `.encryption_key` 文件读取或生成新密钥。

#### 1.5 初始化数据库

```bash
alembic upgrade head
```

此命令会创建 SQLite 数据库文件 `expert_room.db` 并初始化表结构。

#### 1.6 启动后端服务

```bash
uvicorn app.main:app --reload --port 8000
```

启动成功后会显示：

```
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Started reloader process
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Starting Expert Room API version=0.1.0
INFO:     Application startup complete.
```

> **提示**：`--reload` 参数会在代码修改后自动重启，仅用于开发环境。

### 第二步：启动前端服务

#### 2.1 进入前端目录

打开**新的终端窗口**：

```bash
cd project/frontend
```

#### 2.2 安装 Node.js 依赖

```bash
npm install
```

如果安装缓慢，可使用国内镜像源：

```bash
npm config set registry https://registry.npmmirror.com
npm install
```

#### 2.3 启动前端开发服务器

```bash
npm run dev
```

启动成功后会显示：

```
  VITE v5.4.0  ready in 500 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

## 首次使用配置

### 1. 配置 LLM Provider

首次使用需要配置至少一个 LLM 服务提供商：

1. 点击左侧菜单 **「设置」**
2. 点击 **「添加 Provider」**
3. 填写配置信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| **名称** | Provider 名称 | OpenAI |
| **类型** | 服务提供商类型 | OpenAI |
| **Base URL** | API 地址 | `https://api.openai.com/v1` |
| **API Key** | 您的 API 密钥 | `sk-...` |
| **默认模型** | 使用的模型 | `gpt-4o` |

4. 点击 **「测试连接」** 验证配置
5. 测试成功后点击 **「保存」**

### 2. 创建讨论房间

1. 点击左侧菜单 **「创建房间」**
2. 填写房间信息：
   - **房间名称**：描述讨论主题
   - **讨论目标**：详细说明要解决的问题
   - **工作模式**：选择 `code_document` / `document` / `code`
   - **轮次限制**：建议 3-5 轮
   - **输出目录**：产物保存路径
3. 选择参与的 **专家角色卡**（至少 2 个）
4. 为每个专家选择 **Provider**
5. 点击 **「创建房间」**

### 3. 添加数据源（可选）

在房间详情页可以添加讨论上下文：

- **上传文件**：支持 `.txt`, `.md`, `.json`, `.csv`, `.py`, `.ts`, `.js` 等
- **添加文件夹**：自动扫描目录内容
- **添加文本**：直接输入文本内容

### 4. 开始讨论

1. 在房间详情页点击 **「开始讨论」**
2. 观察实时讨论流：
   -   主持人控制讨论节奏
   -   各专家轮流发言
   -   显示当前轮次和进度
3. 讨论完成后自动生成 **产物文档**

### 5. 查看产物

讨论完成后：

1. 在房间详情页查看 **「产物」** 标签
2. 点击产物查看内容
3. 产物文件保存在您指定的 **输出目录** 中

## 常见问题

### Q1: 后端启动报错 `ModuleNotFoundError`

**原因**：Python 虚拟环境未激活或依赖未安装

**解决**：
```bash
# 确保虚拟环境已激活
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

# 重新安装依赖
pip install -r requirements.txt
```

### Q2: 前端启动报错 `ECONNREFUSED`

**原因**：后端服务未启动

**解决**：确保后端服务在 `http://localhost:8000` 正常运行

### Q3: 数据库初始化失败

**原因**：数据库文件被占用或权限问题

**解决**：
```bash
# 删除旧数据库文件（如果有）
# Windows:
del expert_room.db
# macOS/Linux:
rm expert_room.db

# 重新初始化
alembic upgrade head
```

### Q4: API 密钥测试连接失败

**可能原因**：
1. API Key 无效或过期
2. Base URL 不正确
3. 网络代理问题

**解决**：
1. 检查 API Key 是否正确
2. 确认 Base URL 格式（通常以 `/v1` 结尾）
3. 如需代理，在 `.env` 中配置 `HTTP_PROXY` 和 `HTTPS_PROXY`

### Q5: 端口被占用

**后端端口 8000 被占用**：
```bash
# Windows:
netstat -ano | findstr :8000
taskkill /PID <进程ID> /F

# macOS/Linux:
lsof -i :8000
kill -9 <进程ID>
```

**前端端口 5173 被占用**：
Vite 会自动尝试下一个可用端口，或手动指定：
```bash
npm run dev -- --port 5174
```

### Q6: 如何重置所有数据？

```bash
# 1. 删除数据库
cd project/backend
# Windows:
del expert_room.db
# macOS/Linux:
rm expert_room.db

# 2. 删除上传文件
# Windows:
rmdir /s /q uploads
# macOS/Linux:
rm -rf uploads

# 3. 重新初始化
alembic upgrade head
```
