## 更新前准备

### 1. 备份数据

**更新前必须备份重要数据**：

```bash
# 备份数据库
cp project/backend/expert_room.db project/backend/expert_room.db.backup.$(date +%Y%m%d)

# 备份配置文件
cp project/backend/.env project/backend/.env.backup.$(date +%Y%m%d)

# 备份上传文件（如果有）
tar -czf uploads_backup_$(date +%Y%m%d).tar.gz project/backend/uploads
```

### 2. 查看当前版本

```bash
cd project
git log --oneline -5
git describe --tags  # 如果有标签
```

### 3. 查看更新日志

```bash
# 查看远程更新
git fetch origin
git log HEAD..origin/main --oneline
```

### 4. 通知用户（生产环境）

如果在生产环境更新，建议：
- 提前通知用户维护时间
- 选择流量低峰期进行更新
- 准备用户回滚方案

## 本地开发环境更新

### 步骤 1：获取最新代码

```bash
cd project

# 暂存本地修改（如果有）
git stash

# 获取最新代码
git pull origin main

# 恢复暂存的修改（如果有）
git stash pop
```

### 步骤 2：更新后端

```bash
cd backend

# 激活虚拟环境
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

# 更新依赖
pip install -r requirements.txt

# 运行数据库迁移
python -c "from app.database import init_db; import asyncio; asyncio.run(init_db())"

# 验证后端服务
uvicorn app.main:app --reload --port 8000
```

按 `Ctrl+C` 停止测试。

### 步骤 3：更新前端

```bash
cd frontend

# 清除旧的构建文件（可选）
# Windows:
rmdir /s /q dist node_modules
# macOS/Linux:
rm -rf dist node_modules

# 安装依赖
npm install

# 构建生产版本（如果需要）
npm run build

# 启动开发服务器
npm run dev
```

### 步骤 4：验证更新

1. 访问 http://localhost:5173
2. 检查页面是否正常加载
3. 测试核心功能：
   - Provider 连接测试
   - 创建讨论房间
   - 发起讨论

## Linux 服务器更新

### 步骤 1：备份（自动执行）

```bash
# SSH 连接到服务器
ssh user@your-server

# 备份数据库
cp /opt/expert-room/project/backend/expert_room.db /opt/expert-room/backups/expert_room.db.backup.$(date +%Y%m%d)

# 备份配置文件
cp /opt/expert-room/project/backend/.env /opt/expert-room/backups/.env.backup.$(date +%Y%m%d)

# 备份前端文件
cp -r /var/www/expert-room /var/www/expert-room.backup.$(date +%Y%m%d)
```

### 步骤 2：停止服务

```bash
# 停止后端服务
sudo systemctl stop expert-room
```

### 步骤 3：获取最新代码

```bash
cd /opt/expert-room

# 暂存本地修改（如果有）
git stash

# 获取最新代码
git pull origin main

# 恢复暂存的修改（如果有）
git stash pop
```

### 步骤 4：更新后端

```bash
cd /opt/expert-room/project/backend

# 激活虚拟环境
source .venv/bin/activate

# 更新依赖
pip install -r requirements.txt

# 运行数据库迁移
python -c "from app.database import init_db; import asyncio; asyncio.run(init_db())"
```

### 步骤 5：更新前端

```bash
cd /opt/expert-room/project/frontend

# 安装依赖
npm install

# 构建生产版本
npm run build

# 部署到 Nginx
sudo rm -rf /var/www/expert-room/*
sudo cp -r dist/* /var/www/expert-room/
sudo chown -R www-data:www-data /var/www/expert-room
```

### 步骤 6：重启服务

```bash
# 重启后端服务
sudo systemctl start expert-room

# 重新加载 Nginx（如果配置有变更）
sudo nginx -t && sudo systemctl reload nginx
```

### 步骤 7：验证更新

```bash
# 检查服务状态
sudo systemctl status expert-room

# 测试健康接口
curl http://localhost:8000/api/health

# 查看最近日志
sudo journalctl -u expert-room -n 20
```

访问网站验证功能是否正常。

## 数据库迁移

### 查看迁移状态

```bash
cd project/backend
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# 查看当前数据库版本
python -c "from app.database import engine; import asyncio; async def check(): async with engine.begin() as conn: result = await conn.execute('SELECT name FROM sqlite_master WHERE type=\"table\"'); print([r[0] for r in result]); asyncio.run(check())"
```

### 执行迁移

```bash
# 初始化数据库（创建表）
python -c "from app.database import init_db; import asyncio; asyncio.run(init_db())"
```

### 迁移失败处理

如果迁移失败：

```bash
# 查看详细错误
python -c "from app.database import init_db; import asyncio; asyncio.run(init_db())" 2>&1

# 手动修复数据库（如果需要）
# 然后标记当前版本
python -c "from app.database import Base, engine; import asyncio; async def stamp(): async with engine.begin() as conn: await conn.run_sync(Base.metadata.create_all); asyncio.run(stamp())"
```

**注意**：使用 `create_all()` 只会创建不存在的表，不会修改已有表结构。

## 回滚操作

### 步骤 1：代码回滚

```bash
cd project

# 查看提交历史
git log --oneline -10

# 回滚到指定提交
git reset --hard <commit-hash>

# 或回滚到上一个版本
git reset --hard HEAD~1
```

### 步骤 2：数据库回滚

```bash
cd project/backend
source .venv/bin/activate

# 恢复数据库备份
cp /opt/expert-room/backups/expert_room.db.backup.YYYYMMDD /opt/expert-room/project/backend/expert_room.db
```

### 步骤 3：完整回滚流程（生产环境）

```bash
# 1. 停止服务
sudo systemctl stop expert-room

# 2. 恢复代码
cd /opt/expert-room
git reset --hard <old-commit-hash>

# 3. 恢复数据库
cp /opt/expert-room/backups/expert_room.db.backup.YYYYMMDD /opt/expert-room/project/backend/expert_room.db

# 4. 恢复前端
sudo rm -rf /var/www/expert-room/*
sudo cp -r /var/www/expert-room.backup.YYYYMMDD/* /var/www/expert-room/

# 5. 恢复配置文件（如果修改过）
cp /opt/expert-room/backups/.env.backup.YYYYMMDD /opt/expert-room/project/backend/.env

# 6. 重启服务
sudo systemctl start expert-room
sudo systemctl reload nginx
```

## 更新检查清单

更新完成后，逐项检查：

### 后端检查

- [ ] 服务正常启动：`sudo systemctl status expert-room`
- [ ] 健康接口响应：`curl http://localhost:8000/api/health`
- [ ] 数据库版本正确：检查表结构是否完整
- [ ] 无异常日志：`sudo journalctl -u expert-room -n 50`

### 前端检查

- [ ] 页面正常加载
- [ ] 静态资源正常（CSS、JS、图片）
- [ ] API 请求正常（通过浏览器 Network 面板查看）

### 功能检查

- [ ] Provider 连接测试
- [ ] 角色卡片管理
- [ ] 房间创建
- [ ] 讨论功能
- [ ] 产出物生成

### 性能检查

- [ ] 页面加载速度正常
- [ ] API 响应时间正常
- [ ] 无内存泄漏（使用 `htop` 查看）

## 常见问题

### Q1: 更新后依赖安装失败

**原因**：新版本依赖冲突或缓存问题

**解决**：
```bash
# 清除缓存
pip cache purge  # Python
npm cache clean --force  # Node.js

# 删除虚拟环境重新安装
# Python:
rm -rf .venv
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# Node.js:
rm -rf node_modules package-lock.json
npm install
```

### Q2: 数据库迁移失败

**原因**：新版本模型变更与数据冲突

**解决**：
```bash
# 查看详细错误
python -c "from app.database import init_db; import asyncio; asyncio.run(init_db())" 2>&1

# 手动修复数据（如果需要）
# 然后重新初始化
python -c "from app.database import Base, engine; import asyncio; async def fix(): async with engine.begin() as conn: await conn.run_sync(Base.metadata.create_all); asyncio.run(fix())"
```

**注意**：`create_all()` 只创建不存在的表，不会修改已有表。

### Q3: 前端构建失败

**原因**：TypeScript 类型错误或依赖问题

**解决**：
```bash
# 查看详细错误
npm run build

# 修复类型错误后重新构建
# 或使用以下命令检查类型
npx tsc --noEmit
```

### Q4: 更新后服务异常

**原因**：API 接口变更或数据格式变化

**解决**：
1. 查看后端日志了解具体错误
2. 检查 API 文档
3. 清除浏览器缓存后重试

### Q5: 服务器更新后无法访问

**原因**：配置文件或端口冲突

**解决**：
```bash
# 查看详细错误日志
sudo journalctl -u expert-room -n 100

# 检查端口占用
sudo lsof -i :8000

# 检查配置文件
cat /opt/expert-room/project/backend/.env
```

### Q6: 前端页面显示旧版本

**原因**：浏览器缓存或前端文件未更新

**解决**：
1. 强制刷新浏览器：`Ctrl + Shift + R` (Windows/Linux) 或 `Cmd + Shift + R` (macOS)
2. 清除浏览器缓存
3. 确认前端文件已更新：
   ```bash
   ls -la /var/www/expert-room/assets/
   ```

## 自动更新脚本（可选）

创建自动更新脚本：

```bash
nano /opt/expert-room/update.sh
```

脚本内容：

```bash
#!/bin/bash
set -e  # 遇到错误立即退出

echo "=== 开始更新专家团 ==="

# 配置
APP_DIR="/opt/expert-room"
BACKUP_DIR="/opt/expert-room/backups"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 1. 备份
echo "1. 备份数据..."
cp $APP_DIR/project/backend/expert_room.db $BACKUP_DIR/expert_room_$DATE.db
cp $APP_DIR/project/backend/.env $BACKUP_DIR/env_$DATE.backup

# 2. 停止服务
echo "2. 停止服务..."
sudo systemctl stop expert-room

# 3. 更新代码
echo "3. 更新代码..."
cd $APP_DIR
git stash || true
git pull origin main
git stash pop || true

# 4. 更新后端
echo "4. 更新后端..."
cd $APP_DIR/project/backend
source .venv/bin/activate
pip install -r requirements.txt
python -c "from app.database import init_db; import asyncio; asyncio.run(init_db())"

# 5. 更新前端
echo "5. 更新前端..."
cd $APP_DIR/project/frontend
npm install
npm run build
sudo rm -rf /var/www/expert-room/*
sudo cp -r dist/* /var/www/expert-room/
sudo chown -R www-data:www-data /var/www/expert-room

# 6. 重启服务
echo "6. 重启服务..."
sudo systemctl start expert-room
sudo systemctl reload nginx

# 7. 验证
echo "7. 验证服务..."
sleep 3
if curl -s http://localhost:8000/api/health | grep -q "ok"; then
    echo "=== 更新成功 ==="
else
    echo "=== 警告：服务可能未正常启动，请检查日志 ==="
    sudo journalctl -u expert-room -n 20
fi

echo "备份位置: $BACKUP_DIR"
```

添加执行权限：

```bash
chmod +x /opt/expert-room/update.sh
```

使用脚本：

```bash
sudo /opt/expert-room/update.sh
```
