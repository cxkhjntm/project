## 快速启动

### Windows 用户（推荐）

**前提条件**：已安装 Anaconda/Miniconda 并创建了 `Test` 环境。

```bash
# 创建 Conda 环境（如果尚未创建）
conda create -n Test python=3.11 -y
conda activate Test

# 进入项目目录
cd project

# 双击运行启动脚本
start_windows.bat
```

启动脚本会自动：
1. 激活 Conda `Test` 环境
2. 安装后端 Python 依赖
3. 创建配置文件（如果不存在）
4. 初始化数据库
5. 安装前端 Node.js 依赖
6. 在新窗口中启动后端和前端服务

### Linux 用户

```bash
# 进入项目目录
cd project

# 添加执行权限
chmod +x start_linux.sh

# 运行启动脚本
./start_linux.sh
```

启动脚本会自动：
1. 检查 Python 和 Node.js 环境
2. 创建 Python 虚拟环境
3. 安装所有依赖
4. 初始化数据库
5. 在后台启动后端和前端服务

停止服务：
```bash
./stop_linux.sh
```

### macOS 用户

```bash
# 进入项目目录
cd project/backend

# 创建虚拟环境
python3 -m venv .venv
source .venv/bin/activate

# 安装后端依赖
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件，配置 ENCRYPTION_KEY

# 初始化数据库
python -c "from app.database import init_db; import asyncio; asyncio.run(init_db())"

# 启动后端（新终端）
uvicorn app.main:app --reload --port 8000

# 启动前端（另一个终端）
cd ../frontend
npm install
npm run dev
```

## 详细启动步骤

### 第一步：启动后端服务

#### 1.1 进入后端目录

```bash
cd project/backend
```

#### 1.2 创建 Python 虚拟环境

```bash
# 创建虚拟环境
python -m venv .venv

# 激活虚拟环境
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate
```

激活后终端会显示 `(.venv)` 前缀。

#### 1.3 安装 Python 依赖

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

#### 1.4 配置环境变量

```bash
# 复制配置模板
# Windows:
copy .env.example .env
# macOS/Linux:
cp .env.example .env
```

编辑 `.env` 文件，**必须配置** `ENCRYPTION_KEY`：

```bash
# 生成加密密钥（在终端执行）：
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

将生成的密钥填入 `.env` 文件：

```env
ENCRYPTION_KEY=你生成的密钥字符串
```

> **重要**：
> - 此密钥用于加密 LLM API 密钥，保护数据安全。密钥丢失后已保存的 API 密钥将无法解密。
> - 密钥必须是有效的 Fernet 格式（32 字节 URL-safe Base64 编码，通常为 44 个字符，以 `=` 结尾）。

#### 1.5 初始化数据库

```bash
python -c "from app.database import init_db; import asyncio; asyncio.run(init_db())"
```

#### 1.6 启动后端服务

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

启动成功后会显示：

```
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
INFO:     Application startup complete.
```

### 第二步：启动前端服务

在**新的终端窗口**中执行：

#### 2.1 进入前端目录

```bash
cd project/frontend
```

#### 2.2 安装 Node.js 依赖

```bash
npm install
```

#### 2.3 启动前端开发服务器

```bash
npm run dev
```

启动成功后会显示：

```
  VITE v5.4.0  ready in 500 ms

  ➜  Local:   http://localhost:5173/
```

### 第三步：访问应用

在浏览器中打开：**http://localhost:5173**

## 首次使用引导

### 1. 配置 LLM Provider

首次使用需要先配置至少一个 LLM 服务提供商：

1. 点击左侧菜单 **「设置」**
2. 点击 **「添加新 Provider」**
3. 填写配置信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| **名称** | Provider 名称 | OpenAI |
| **类型** | 服务提供商类型 | OpenAI |
| **Base URL** | API 地址 | `https://api.openai.com/v1` |
| **API Key** | 你的 API 密钥 | `sk-...` |
| **默认模型** | 使用的模型 | `gpt-4o` |

4. 点击 **「测试连接」** 验证配置
5. 测试成功后点击 **「保存」**

### 2. 创建讨论房间

1. 点击左侧菜单 **「专家群聊室」**
2. 填写房间信息：
   - **房间名称**：描述本次讨论主题
   - **讨论目标**：详细说明要讨论的内容
   - **讨论模式**：选择 `code_document` / `document` / `code`
   - **执行轮数**：建议 3-5 轮
   - **产出目录**：文件保存路径
3. 选择参与的 **专家角色**（至少 2 个）
4. 为每个专家选择 **Provider**
5. 点击 **「创建群聊室」**

### 3. 添加数据源（可选）

在房间详情页面，可以添加讨论素材：

- **上传文件**：支持 `.txt`, `.md`, `.json`, `.csv`, `.py`, `.ts`, `.js` 等
- **指定文件夹**：自动扫描目录内容
- **粘贴文本**：直接粘贴文本内容

### 4. 开始讨论

1. 在房间详情页点击 **「开始讨论」**
2. 观察实时讨论过程：
   -  主持人引导讨论方向
   -  专家轮流发言
   - 显示当前轮次和进度
3. 讨论完成后自动生成 **产出文档**

### 5. 查看产出

讨论完成后：

1. 在房间详情页查看 **「产出」** 标签
2. 预览产出内容
3. 文件已保存到指定的 **产出目录**

## 常见问题

### Q1: 启动时报错 `ModuleNotFoundError`

**原因**：Python 虚拟环境未激活或依赖未安装

**解决**：
```bash
# 确保虚拟环境已激活
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

# 重新安装依赖
pip install -r requirements.txt
```

### Q2: 前端启动报错 `ECONNREFUSED`

**原因**：后端服务未启动

**解决**：确保后端服务在 `http://localhost:8000` 正常运行

### Q3: 数据库初始化失败

**原因**：数据库文件被占用或权限问题

**解决**：
```bash
# 删除旧数据库文件（如果存在）
# Windows:
del expert_room.db
# macOS/Linux:
rm expert_room.db

# 重新初始化
python -c "from app.database import init_db; import asyncio; asyncio.run(init_db())"
```

### Q4: API 连接测试失败

**常见原因**：
1. API Key 无效或过期
2. Base URL 不正确
3. 网络连接问题

**解决**：
1. 检查 API Key 是否正确
2. 确认 Base URL 格式（通常以 `/v1` 结尾）
3. 检查网络连接，在 `.env` 中配置 `HTTP_PROXY` 和 `HTTPS_PROXY`

### Q5: 端口被占用

**后端端口 8000 被占用**：
```bash
# Windows:
netstat -ano | findstr :8000
taskkill /PID <进程ID> /F

# macOS/Linux:
lsof -i :8000
kill -9 <进程ID>
```

**前端端口 5173 被占用**：
Vite 会自动尝试下一个可用端口，或手动指定：
```bash
npm run dev -- --port 5174
```

### Q6: 如何重置所有数据

```bash
# 1. 删除数据库
cd project/backend
# Windows:
del expert_room.db
# macOS/Linux:
rm expert_room.db

# 2. 删除上传文件
# Windows:
rmdir /s /q uploads
# macOS/Linux:
rm -rf uploads

# 3. 重新初始化
python -c "from app.database import init_db; import asyncio; asyncio.run(init_db())"
```
