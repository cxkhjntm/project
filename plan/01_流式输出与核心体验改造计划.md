# 01. 流式输出与核心体验改造计划 (详细)

## 1. 改造模型客户端 (`backend/app/services/model_client.py`)
目前的 `ModelClient` 只有同步阻塞的 `chat_completion` 方法。我们需要补充 `chat_completion_stream` 方法，利用 HTTPX 的异步流读取 SSE 格式。

**具体代码实现：**
```python
import json

async def chat_completion_stream(
    self,
    messages: List[Dict[str, str]],
    temperature: Optional[float] = None,
    max_tokens: Optional[int] = None,
    stop: Optional[List[str]] = None,
):
    payload = {
        "model": self.model,
        "messages": messages,
        "temperature": temperature or self.temperature,
        "max_tokens": max_tokens or self.max_tokens,
        "stream": True,  # 关键：开启 OpenAI 标准流式输出
    }
    if stop:
        payload["stop"] = stop
    
    headers = {
        "Authorization": f"Bearer {self.api_key}",
        "Content-Type": "application/json",
    }
    
    async with httpx.AsyncClient(timeout=self.timeout) as client:
        async with client.stream(
            "POST",
            f"{self.base_url}/chat/completions",
            headers=headers,
            json=payload,
        ) as response:
            if response.status_code != 200:
                error_text = await response.aread()
                raise ModelClientError(f"API Error {response.status_code}: {error_text.decode('utf-8')}")
            
            async for line in response.aiter_lines():
                if line.startswith("data: "):
                    line_data = line[6:]
                    if line_data.strip() == "[DONE]":
                        break
                    try:
                        data = json.loads(line_data)
                        delta = data["choices"][0].get("delta", {})
                        if "content" in delta:
                            yield delta["content"]
                    except json.JSONDecodeError:
                        logger.error(f"Error parsing stream line: {line}")
```

## 2. 改造编排器发射事件 (`backend/app/services/orchestrator.py`)
将原来等待完整回答后才发出的机制，改为在 `async for` 循环中密集 `emit` 增量 token。

**具体修改步骤：**
1. 在 `SSEEventType` 枚举中新增 `TOKEN = "token"`。
2. 修改 `_run_expert_turn` 方法中的调用方式：
```python
# 原来的代码:
# response = await client.chat_completion(messages=[...])

# 改造后的代码:
full_content = ""
async for chunk in client.chat_completion_stream(messages=[{"role": "user", "content": prompt}]):
    full_content += chunk
    # 每次获取 chunk 立即推送给前端
    await self.emit_event(SSEEventType.TOKEN, {
        "room_id": self.room_id,
        "role": role_name,
        "content": chunk,
    })

# 循环结束后，再将完整消息存入数据库，并发出完整的 MESSAGE 事件
message_data = MessageCreate(
    room_id=self.room_id,
    sender_type="expert",
    sender_id=role_card_id,
    content=full_content, # 使用拼装好的全量内容
    round=self.current_round,
)
```

## 3. 前端消费端适配 (`frontend/src/hooks/useDiscussionSSE.ts`)
前端需要拦截 `token` 事件，维护一个按角色分类的正在生成的字符缓冲（`streamingMessage`）。

**具体修改步骤：**
1. 在 Hook 状态中新增 `streamingMessages`：
   ```typescript
   const [streamingMessages, setStreamingMessages] = useState<Record<string, string>>({});
   ```
2. 增加对 `token` 事件的监听：
   ```typescript
   eventSource.addEventListener('token', (event) => {
     try {
       const data = JSON.parse(event.data);
       setStreamingMessages((prev) => ({
         ...prev,
         [data.role]: (prev[data.role] || '') + data.content
       }));
     } catch (e) {
       console.error('Parse token error:', e);
     }
   });
   ```
3. 在收到最终 `message` 事件时，清理对应的缓冲流：
   ```typescript
   eventSource.addEventListener('message', (event) => {
       // ...原有添加 messages 逻辑...
       // 收到完整消息后，清空正在生成的流状态
       setStreamingMessages((prev) => {
           const next = { ...prev };
           // 获取发言角色名字来清理
           const senderName = data.sender_id === null ? "主持人" : participantNameMap[data.sender_id];
           if (senderName) delete next[senderName];
           return next;
       });
   });
   ```
