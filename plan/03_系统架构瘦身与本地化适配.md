# 03. 系统架构瘦身与本地化适配 (详细)

由于项目本质是供个人程序员在本地高效使用的工具，原作者为了迎合企业级或通用部署标准引入的某些设计显得冗余。我们将做针对性降级。

## 1. 移除 Alembic，启用轻量化自动建表
**目标文件**: `backend/app/main.py`
个人使用 SQLite 的好处就是随时可以通过 DB Browser 直接删库重建或者手动补字段。运行冗长的 Alembic 生成指令完全没必要。

**修改步骤：**
1. 删除项目中的 `alembic/` 文件夹及 `alembic.ini` 文件。
2. 在 `main.py` 的 `startup_event` 钩子中增加自动建表逻辑：
```python
from app.database import engine
from app.models.base import Base # 确保引入了所有的 ORM 模型

@app.on_event("startup")
async def startup_event() -> None:
    logger.info("Starting Expert Room API")
    
    # 新增：启动时自动依据 SQLAlchemy 定义在 SQLite 中建表
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
        
    async with async_session_factory() as session:
        await load_builtin_roles(session)
        await session.commit()
```

## 2. API 密钥免密/明文存储适配
**目标文件**: `backend/app/config.py` 和 `backend/app/services/crypto.py`
防止在换电脑或误删 `.encryption_key` 后无法使用。

**修改步骤：**
1. `config.py` 增加开关：
```python
class Settings(BaseSettings):
    # ...
    # 默认对于个人项目可以设置为 False
    encrypt_api_keys: bool = False
```
2. `crypto.py` 改造加解密绕过：
```python
from app.config import settings

class CryptoService:
    def encrypt(self, data: str) -> str:
        if not settings.encrypt_api_keys:
            return data # 明文直存
        return self.fernet.encrypt(data.encode()).decode()

    def decrypt(self, encrypted_data: str) -> str:
        if not settings.encrypt_api_keys:
            return encrypted_data # 明文直读
        # 如果历史数据是加密的，尝试解密
        try:
            return self.fernet.decrypt(encrypted_data.encode()).decode()
        except Exception:
            # 万一读到了明文却尝试解密失败，直接返回原值（兼容降级）
            return encrypted_data
```

## 3. 复用 HTTPX 连接池 (提升并发性能)
**目标文件**: `backend/app/services/model_client.py`
为了避免每次专家发言（甚至是拉取模型测试）都产生全新的 TCP 连接，我们应当使用全局 Http Client。

**修改步骤：**
```python
# 在 model_client.py 顶部初始化一个全局 client
_global_async_client = httpx.AsyncClient(timeout=120.0)

# 在 ModelClient 类中使用这个全局实例：
async def chat_completion(self, ...):
    # 替换原来的 `async with httpx.AsyncClient(...) as client:` 
    response = await _global_async_client.post(
        f"{self.base_url}/chat/completions",
        headers=headers,
        json=payload,
    )
    # ...
```
> *注：若采用此方案，需在 `main.py` 的 `shutdown_event` 中执行 `await _global_async_client.aclose()` 以优雅关闭。*

## 4. 本地文件绝对路径直读支持
目前 `backend/app/routers/filesystem.py` 支持浏览目录，但加载上下文时依然受限于文件大小限制。
对于个人版，既然前后端跑在同一台物理机，后端具有完整磁盘权限，可直接在 `app/services/file_ingestion.py` 中判断如果是“本地直读”模式，直接越过所有上传校验：

```python
async def ingest_local_file(filepath: str) -> str:
    """新增加的直读支持"""
    import aiofiles
    # 个人使用，无视 10MB 限制直接硬读
    async with aiofiles.open(filepath, mode='r', encoding='utf-8', errors='replace') as f:
        return await f.read()
```
这让用户通过前端文件树点击任何复杂的代码工程文件，瞬间无缝装载到 Shared Sources 中。
